# 容器父节点选择修复与小幅精简设计

## 目标

修复容器编辑页的父容器选择范围过窄问题，并做一次小幅代码整理，保持改动尽量局部。

## 问题

当前 `js/app.js` 的容器编辑页只从根容器中生成父容器候选项，导致多层容器结构下无法把容器改挂到其他合法父容器下。这个问题和“无限层级容器”的产品能力不一致。

## 设计

### 1. 父容器候选列表

新增一个小型 helper，用于生成“可选父容器”列表：

- 输入：当前容器 id
- 输出：所有可作为父容器的容器
- 规则：排除当前容器本身，以及它的所有后代，避免形成循环

容器编辑页直接使用这个 helper 渲染下拉框，不再只依赖根容器列表。

### 2. 代码精简

把父容器筛选逻辑独立出来，减少 `renderContainerEdit()` 里的分支和数据库查询意图混杂的问题。

### 3. 数据流

`renderContainerEdit()` -> 获取可选父容器 -> 渲染 `<select>` -> 保存时仍按原逻辑写回 `parentId`。

## 错误处理

- 如果可选父容器列表为空，只保留“顶层（无父容器）”选项。
- 如果扫描/保存路径中出现现有异常，维持现有提示方式，不新增吞错。

## 测试

- 手动验证：
  - 新建多层容器后，编辑子容器时能看到合法的更深层父容器候选。
  - 不能把容器设置成自己或自己的后代。
  - 保存后容器树和详情页正常显示。
- 代码层面检查：
  - 确认 `js/app.js` 无语法错误。
  - 确认修改不影响物品页、扫码页和提醒页。
